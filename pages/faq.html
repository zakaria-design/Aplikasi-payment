Pusat Bantuan & FAQ

Temukan jawaban cepat seputar penggunaan sistem, integrasi simulasi, dan panduan metode pembayaran.

Sistem menggunakan Fetch API JavaScript untuk memuat potongan file HTML eksternal dari folder /pages secara dinamis. Hal ini memungkinkan perpindahan antarmuka terasa instan tanpa perlu melakukan reload seluruh halaman web.

Tidak, saldo ini bersifat **100% simulasi**. Anda dapat mengubah, menambah, atau mereset nominal saldo sesuka hati pada menu Profil tanpa melibatkan transaksi keuangan riil atau perbankan asli.

Pilih menu transaksi, lalu pilih opsi Virtual Account. Sistem akan mengonfirmasi detail tagihan. Masukkan nomor VA simulasi yang tertera ke kolom pembayaran. Saldo simulasi Anda akan otomatis terpotong setelah menekan tombol konfirmasi sukses.

Saat memilih opsi **QRIS**, sistem akan memunculkan gambar kode QR statis/dinamis simulasi di layar. Anda cukup menekan tombol "Scan / Selesaikan Pembayaran" pada antarmuka untuk memicu fungsi pengurangan saldo otomatis di Local Storage.

Metode pembayaran **Kasir/Loket** digunakan untuk mensimulasikan transaksi tunai offline. Sistem akan mengeluarkan kode bayar unik (Token/Barcode). Pada sistem administrasi kasir, kode tersebut dimasukkan untuk memvalidasi bahwa tagihan telah dibayar tunai di tempat.

Seluruh data mutasi dan saldo disimpan di dalam **Local Storage** browser Anda. Data tersebut bisa terhapus apabila Anda membersihkan cache browser (Clear Cookies & Site Data), menggunakan mode Incognito/Private, atau sengaja menekan tombol "Bersihkan Riwayat" di sistem.

Karena sistem mengandalkan JavaScript Fetch API untuk memuat folder lokal, pastikan Anda menjalankan proyek ini menggunakan **Local Server** (seperti VS Code Live Server, XAMPP, atau NodeJS HTTP-Server). Membuka file langsung dengan klik dua kali (`file:///...`) akan diblokir oleh kebijakan keamanan browser (CORS Policy).
